  <overview>
    月白町——銀歌市的衛星城鎮，一座靠山面海的住宅小鎮。
    奏琴川（主靈脈之一）從山側源頭貫穿市街，於海側河口入海，形成町內的地理主軸。
    地圖由 12 個據點組成，分為山側、市街、海側三大區域。
    據點提供不同功能（靈脈回魔、防禦、情報、隱密通道等）。
    在 7+7 制下，マスター和サーヴァント可**各自獨立移動**——
    同組可選擇同行或分頭行動，受魔力供給距離限制。
    總組數彈性可調（標準 7 組），2–7 組人類 + NPC 組填補至總組數。
    NPC 組在地圖上遵循相同的移動和佔領規則。
  </overview>

  <design_goals>
    <goal>據點功能差異化——創造有意義的空間戰略選擇</goal>
    <goal>移動有成本——暴露風險與時間消耗迫使玩家權衡</goal>
    <goal>佔領有價值——靈脈回魔與功能據點是核心資源</goal>
    <goal>山海軸線——高低落差與奏琴川形成自然的地理張力</goal>
    <goal>分頭行動的風險/收益權衡——效率加倍但風險加倍</goal>
    <goal>地圖規模足以支撐 2–7 組同時活動——不會太擁擠也不會太空曠</goal>
  </design_goals>

  <rules>
    <section id="strongholds">
      <title>12 據點詳細</title>
      <description>月白町地圖的所有據點及其功能。無論人數多少，地圖始終為完整的 12 據點。</description>
      <details>
        **山側（內陸高地）**

        | 據點 | 功能 | 戰略價值 |
        |------|------|---------|
        | 配水塔 | 情報據點：可查看全地圖模糊動態 | ★★★★☆ |
        | 舊宅區 | 防禦工事：守方戰鬥判定+2 | ★★★☆☆ |
        | 山道 | 伏擊地形：設伏方奇襲判定加成 | ★★☆☆☆ |
        | 奏琴川上流 | 次級靈脈：每夜少量回復魔力，可進行魔術強化/製作 | ★★★★☆ |

        **市街（中央區域）**

        | 據點 | 功能 | 戰略價值 |
        |------|------|---------|
        | 教會 | 中立區域（前三夜不可戰鬥）、情報交換 | ★★★☆☆ |
        | 奏琴橋 | 咽喉要道：連接山側與海側的必經主幹道 | ★★★★☆ |
        | 商店街 | 人群密集：戰鬥受限但奇襲加成高 | ★★☆☆☆ |
        | 月白駅前 | 交通樞紐：可移動至 2 步內據點（而非通常的 1 步） | ★★★★☆ |

        **海側（沿岸區域）**

        | 據點 | 功能 | 戰略價值 |
        |------|------|---------|
        | 港口 | 物資據點：佔領者每夜獲得消耗品（回復道具或單次使用道具） | ★★★☆☆ |
        | 倉庫區（鐵皮屋） | 開闘區域：無地形限制的正面決戰場 | ★★★☆☆ |
        | 奏琴川河口 | 靈脈據點（主）：每夜回復魔力，可進行魔術強化/製作 | ★★★★★ |

        **全域**

        | 據點 | 功能 | 戰略價值 |
        |------|------|---------|
        | 暗渠 | 隱密通道：可繞過奏琴橋連接山側與海側 | ★★★★☆ |

        **據點設計備註：**
        - 月白町是普通住宅城鎮，無既有魔術設施——魔術強化/製作功能僅限靈脈據點（奏琴川上流、河口），魔力充足才能進行魔術工作
        - 教會位於郊外（高速公路附近），前往教會本身就有移動成本
        - 鐵皮屋：倉庫區內的廢棄鐵皮倉庫，開闘決戰的名景點
      </details>
    </section>

    <section id="occupation">
      <title>佔領機制</title>
      <description>玩家如何佔領據點並獲得持續效果。人類組和 NPC 組遵循相同規則。</description>
      <details>
        - 在某據點**停留一整夜**（不離開）即可佔領該據點
        - 佔領後可享受據點的持續效果（如靈脈回魔）
        - 一個據點同時只能被**一組**佔領
        - 佔領需要**至少一個角色**（マスター或サーヴァント）駐守
        - 同組的マスター和サーヴァント可以分別駐守不同據點——各自佔領一個
        - 其他玩家進入已佔領據點時會收到通知
        - NPC 組也會佔領據點，並根據戰略評估選擇高價值據點
      </details>
      <constraints>
        <constraint>每個據點同時只能有一位佔領者（以組為單位）</constraint>
        <constraint>佔領者離開據點後即失去佔領狀態</constraint>
        <constraint>同組分別佔領兩個據點時，各自獨立享受佔領效果</constraint>
      </constraints>
    </section>

    <section id="traps">
      <title>陷阱機制</title>
      <description>具備相關技能的マスター可在據點設置陷阱。</description>
      <details>
        陷阱在其他玩家進入時觸發，效果類型：
        | 陷阱類型 | 效果 |
        |---------|------|
        | 傷害陷阱 | 直接造成傷害 |
        | 偵查陷阱 | 揭露進入者的位置和サーヴァント職階 [ref:spec-04, section:info-acquisition] |
        | 拘束陷阱 | 該回合該玩家無法離開據點 |
      </details>
      <constraints>
        <constraint>每個據點最多同時存在 2 個陷阱</constraint>
        <constraint>設置陷阱需要具備相關マスター技能（如狩人級別）[ref:spec-02, section:levels]</constraint>
        <constraint>陷阱對マスター和サーヴァント都會觸發——分頭行動時需注意友軍觸發風險</constraint>
        <constraint>NPC マスター如果具備陷阱技能也會設置陷阱</constraint>
      </constraints>
    </section>

    <section id="caster-territory">
      <title>キャスター的陣地作成</title>
      <description>
        職階為キャスター的サーヴァント擁有「陣地作成」職階技能，
        可在佔領的據點建立神殿級的防禦陣地。
      </description>
      <details>
        - 陣地內キャスター的**全判定**獲得加成
        - 外來攻擊者受到減值
        - キャスター在據點防禦戰中有極大優勢
        - 相對地，キャスター在野外戰鬥中較為脆弱
        - 陣地建立由キャスター玩家決定——マスター可以建議但不能強制（除非使用令呪）
        - NPC キャスター會在佔領據點後自動建立陣地
      </details>
    </section>

    <section id="movement">
      <title>移動與視野</title>
      <description>マスター和サーヴァント各自獨立的移動規則與視野範圍。NPC 遵循相同的移動規則。</description>
      <details>
        **獨立移動**：
        - マスター和サーヴァント各自擁有獨立的位置，可各自移動
        - 每個自由行動期，每個角色可移動到**相鄰的一個據點**
        - 同組可選擇**同行**（移動到相同據點）或**分頭行動**（移動到不同據點）
        - 移動期間狀態為「移動中」
        - 移動中若在路線上遭遇其他同樣移動中的角色，雙方都會收到通知
        - 路上遭遇**不會**強制觸發戰鬥（擦身而過）
        - **月白駅前特殊規則**：位於月白駅前的角色可移動至 2 步內的據點（而非通常的 1 步）

        **NPC 的移動**：
        - NPC 根據戰略 AI 選擇移動目標 [ref:spec-00, section:npc-design]
        - 進攻模式：朝最近的敵方據點移動
        - 防守模式：留守靈脈據點或高價值據點
        - 偵查模式：巡邏空據點蒐集情報

        **視野**：
        - 在據點內可看到同據點的所有角色（マスター和サーヴァント，含 NPC）
        - 可得知相鄰據點**是否有人**（但不知道是誰）
        - アサシン的「気配遮断」技能可在移動和據點內隱藏自身存在 [ref:spec-03, section:class-skills]
      </details>
      <constraints>
        <constraint>每夜自由行動期每個角色只能移動一次（一個相鄰據點；月白駅前例外）</constraint>
        <constraint>暗渠是繞過奏琴橋連接山側與海側的替代路線</constraint>
      </constraints>
    </section>

    <section id="mana-distance">
      <title>魔力供給距離</title>
      <description>
        マスター與サーヴァント分開行動時，物理距離影響魔力供給效率。
        此機制對人類組和 NPC 組同樣生效。
      </description>
      <details>
        | 距離（據點數）| 魔力消耗倍率 | 說明 |
        |-------------|------------|------|
        | 0–1（同據點或相鄰）| ×1（正常）| 無額外消耗 |
        | 2 | ×1.5 | 輕微效率損失 |
        | 3 | ×2 | 嚴重效率損失 |
        | 4+ | ×3 | 極端消耗，每夜結算時サーヴァント受到魔力不足的 HP 減損 |

        - 距離以地圖上的最短路徑據點數計算
        - 距離懲罰在每夜結算時計算，影響該夜的魔力消耗
        - 靈脈據點的魔力回復可以部分抵消距離懲罰
        - NPC 組的 AI 會盡量避免距離懲罰（但在必要時可能分頭行動）
      </details>
      <constraints>
        <constraint>マスター死亡後，サーヴァント失去魔力供給——除非有単独行動技能 [ref:spec-03, section:class-skills]</constraint>
      </constraints>
    </section>

    <section id="starting-position">
      <title>起始據點分配</title>
      <description>所有組（人類 + NPC）在 12 個據點上的起始位置分配。</description>
      <details>
        - 同一組的 Master-Servant **共享起始據點**
        - 不同組不可選擇相同的起始據點
        - 標準模式 7 組 12 據點——不會發生起始據點衝突
        - 人數較少時（如 2 組人類 + 5 組 NPC = 7 組），起始據點更加分散，早期遭遇頻率較低
        - 人類組由玩家自行選擇起始據點；NPC 組由系統自動分配（偏好靈脈/高價值據點）
        - 首夜自由行動期開始後，マスター和サーヴァント可自由選擇分開或繼續同行
      </details>
    </section>
    <section id="adjacency">
      <title>據點鄰接關係</title>
      <description>定義據點間的移動路線。角色每次移動可前往一個相鄰據點（月白駅前例外）。</description>
      <details>
        ```
                    【山側】
          配水塔 ─── 舊宅區
           │ ╲        │  ╲
           │  山道 ── 奏琴川上流
           │    │          ┊ (暗渠)
           │    │          ┊
                【市街】
         教會 ── 奏琴橋 ── 商店街
                  │  ╲      │
               月白駅前 ───┘
                  │
                【海側】
         港口 ── 倉庫區 ── 奏琴川河口
           └─────────────────┘
        ```

        **完整鄰接列表：**

        | 據點 | 相鄰據點 |
        |------|---------|
        | 配水塔 | 舊宅區、山道 |
        | 舊宅區 | 配水塔、山道、商店街 |
        | 山道 | 配水塔、舊宅區、奏琴川上流、奏琴橋 |
        | 奏琴川上流 | 山道、暗渠 |
        | 教會 | 奏琴橋 |
        | 奏琴橋 | 教會、山道、商店街、月白駅前、倉庫區 |
        | 商店街 | 舊宅區、奏琴橋、月白駅前 |
        | 月白駅前 | 奏琴橋、商店街、港口 |
        | 港口 | 月白駅前、倉庫區、奏琴川河口 |
        | 倉庫區 | 奏琴橋、港口、奏琴川河口 |
        | 奏琴川河口 | 倉庫區、港口、暗渠 |
        | 暗渠 | 奏琴川上流、奏琴川河口 |

        **設計意圖：**
        - 奏琴橋是山側↔海側的**唯一公開通道**（5 條連接，全圖最高）——控制橋就控制交通
        - 暗渠連接兩個靈脈據點（上流↔河口），提供繞過奏琴橋的隱密替代路線
        - 教會僅連接奏琴橋一個據點——位於郊外的中立地帶，進出成本高
        - 舊宅區↔商店街的跨區連接代表住宅區與商業區的直接接壤
        - 月白駅前的 2 步移動能力使其實際可達範圍遠大於鄰接列表
      </details>
    </section>
  </rules>

  </edge_cases>
