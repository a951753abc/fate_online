  <overview>
    サーヴァント是由聖杯召喚的英靈，由另一位玩家扮演（非マスター玩家本人）。
    從者玩家擁有獨立意志、獨立行動權、獨立情報，與マスター之間是締約合作關係而非主從關係。
    每騎サーヴァント由職階、五圍（骰定生成）、職階技能、保有スキル、宝具、聖杯願望六大要素定義。
  </overview>

  <design_goals>
    <goal>隨機骰定五圍帶來每場遊戲的獨特性——相同英靈在不同場次可能有不同強度</goal>
    <goal>職階差異提供戰術多樣性——每個職階有明確的強項與弱點</goal>
    <goal>宝具真名機制重現原作「隱藏底牌」的策略性</goal>
    <goal>從者玩家擁有完整的角色體驗——有自己的秘密、目標、社交空間和決策權</goal>
    <goal>Master-Servant 關係的核心張力來自情報不對稱與願望衝突</goal>
  </design_goals>

  <rules>
    <section id="servant-player">
      <title>從者玩家角色定位</title>
      <description>サーヴァント由獨立的玩家扮演，具有完整的遊戲參與權。</description>
      <details>
        **從者玩家的權限：**
        - 獨立的行動權——可與マスター同行或單獨行動 [ref:spec-05, section:movement]
        - 獨立的情報——從者持有的情報不自動與マスター共享 [ref:spec-04, section:intra-pair]
        - 獨立的社交——可與其他從者、其他マスター私訊溝通 [ref:spec-07, section:channels]
        - 戰鬥中的操作權——從者玩家自行選擇攻擊目標、技能、是否解放宝具 [ref:spec-06, section:actions]
        - 可無視マスター的戰術建議（非令呪命令）

        **從者玩家的限制：**
        - 受令呪約束——マスター消耗令呪時，從者的行動被強制覆蓋 [ref:spec-02, section:command-spells]
        - 魔力依賴——與マスター的物理距離影響魔力供給效率 [ref:spec-05, section:mana-distance]
        - マスター死亡時開始消散（除非擁有単独行動技能）

        **從者玩家知道什麼：**
        - 自己的完整資料（五圍、技能、宝具、真名）
        - 自己的聖杯願望
        - マスター的名稱

        **從者玩家不知道什麼：**
        - マスター的級別、風格、背反律（需マスター主動揭露或透過觀察推測）
        - マスター的聖杯願望
        - 其他サーヴァント的任何資訊（適用情報迷霧系統）
      </details>
    </section>

    <section id="grail-wish">
      <title>聖杯願望</title>
      <description>
        每位サーヴァント持有一個聖杯願望。願望是純敘事元素——系統不判定內容、不評估相容性。
        遊戲機制只管一件事：終局時**誰拿到許願權**。
      </description>
      <details>
        **願望分配：**
        - 每個英靈有預設的聖杯願望（基於原作設定或自定義英靈設定）
        - 遊戲開始時只有從者玩家自己知道自己的願望
        - マスター也有自己的願望 [ref:spec-02, section:master-wish]，雙方互不知曉

        **願望與許願權：**
        - 聖杯只回應一人——終局時マスター和サーヴァント爭奪許願權 [ref:spec-01, section:victory-condition]
        - 從者玩家的勝利 = 自己拿到許願權
        - 即使在原作中，願望也可能被放下或改變——終局選擇「讓渡」代表此可能性

        **從者的終局優勢與劣勢：**
        - 優勢：戰鬥力遠高於マスター——若進入聖杯爭奪戰鬥，從者幾乎必勝
        - 劣勢：マスター持有令呪時可強制從者讓渡（覆蓋型令呪 + 対魔力抵抗判定）
        - 核心策略：全場觀察マスター消耗了幾枚令呪——0 枚時マスター仍有威懾力，但用完後從者即有機會
      </details>
      <constraints>
        <constraint>從者可以對マスター撒謊自己的願望內容</constraint>
        <constraint>願望內容不受系統約束——玩家自由決定，無預設池或分類限制</constraint>
      </constraints>
    </section>

    <section id="classes">
      <title>七騎職階</title>
      <description>基礎七職階及其特點與職階技能。</description>
      <details>
        | 職階 | 特點 | 職階技能 |
        |------|------|---------|
        | セイバー（劍兵）| 高筋力/耐久，近戰主力 | 対魔力、騎乗 |
        | アーチャー（弓兵）| 高敏捷，遠程攻擊 | 単独行動（マスター死亡後仍可獨立參戰）、対魔力 |
        | ランサー（槍兵）| 高敏捷，機動性強 | 対魔力 |
        | ライダー（騎兵）| 均衡型，宝具強力 | 騎乗（移動速度加成）、対魔力 |
        | キャスター（魔術師）| 高魔力，支援與結界 | 陣地作成（據點防禦加成）、道具作成 |
        | アサシン（暗殺者）| 高敏捷/幸運，隱匿 | 気配遮断（隱身 + 奇襲加成）|
        | バーサーカー（狂戰士）| 全能力強化但溝通受限 | 狂化（全能力 +1 等級，溝通限制）|
      </details>
    </section>

    <section id="berserker">
      <title>バーサーカー（狂職）——特殊玩家體驗</title>
      <description>
        狂職サーヴァント由玩家扮演，但因狂化效果而受到嚴格的溝通限制。
        這創造了一種獨特的遊戲體驗：你是最強的戰鬥力，但無法用語言表達自己的意志。
      </description>
      <details>
        **溝通限制：**
        - 無法使用任何文字聊天頻道（公頻、私訊、組內私頻）
        - 只能使用預設的表情符號和簡單指令面板：
          - 表情：怒 / 悲 / 喜 / 殺意 / 困惑 / 忠誠 / 拒絕
          - 指令：攻擊 / 防禦 / 前進 / 後退 / 跟隨 / 等待
        - マスター對狂職說話時，狂職玩家**聽得到**（看到訊息），但只能用表情回應

        **戰鬥操作權：**
        - 狂職玩家保留完整的戰鬥操作權（選擇攻擊目標、使用技能）
        - 宝具真名解放的決定權仍在狂職玩家手上
        - 但狂化的「理智喪失」表現為：戰鬥行動選項中會額外出現「暴走」選項——隨機攻擊場上任何人（含盟友）

        **令呪對狂職的效果：**
        - 令呪是マスター**唯一能精確指揮狂職的手段**
        - 效果不變（強制行動、傳送、宝具增幅）[ref:spec-02, section:command-spells]
        - バーサーカー職階不保有「対魔力」——マスター的覆蓋型令呪無法被抵抗

        **狂職的聖杯願望：**
        - 狂職同樣有聖杯願望，但マスター只能透過觀察狂職的表情反應去猜測
        - 如果マスター猜對並主動協助實現 → 可能觸發特殊的雙贏結局
      </details>
    </section>

    <section id="class-skills">
      <title>職階技能詳細效果</title>
      <description>各職階技能的具體遊戲效果。</description>
      <details>
        | 職階技能 | 效果 |
        |---------|------|
        | 対魔力 | 對魔術攻擊的判定獲得抗性加成。等級 A 以上時，可對マスター的覆蓋型令呪進行抵抗判定 [ref:spec-02, section:command-spells] |
        | 騎乗 | 移動速度加成，可操縱騎乘物 |
        | 単独行動 | マスター死亡後從者玩家可繼續獨立參戰，不受魔力消散限制 [ref:spec-01, section:victory-edge-cases] |
        | 陣地作成 | 在佔領據點建立神殿級防禦陣地，陣地內全判定獲得加成 [ref:spec-05, section:caster-territory] |
        | 道具作成 | 可製作魔術道具（消耗魔力與時間）|
        | 気配遮断 | 隱藏自身存在，移動和據點內不被偵測 [ref:spec-05, section:movement] |
        | 狂化 | 全能力等級 +1，但從者玩家的溝通受到嚴格限制（見 berserker 專節）|

        **各職階の対魔力保有狀況：**

        | 職階 | 対魔力 | 備註 |
        |------|--------|------|
        | セイバー | ✓ | 騎士職階，通常較高 |
        | アーチャー | ✓ | 騎士職階 |
        | ランサー | ✓ | 騎士職階 |
        | ライダー | ✓ | 騎乘職階 |
        | キャスター | ✗ | 魔術職階，無此技能 |
        | アサシン | ✗ | 暗殺職階，無此技能 |
        | バーサーカー | ✗ | 狂化職階，無此技能（但狂化前的原始英靈可能透過保有スキル保有類似能力）|

        - **職階決定「有沒有」対魔力，等級取決於個別英靈**（忠於原作設定）
        - 等級 A 以上時可對覆蓋型令呪進行抵抗判定 [ref:spec-02, section:command-spells]
        - 職階技能等級的決定方式見 open_questions
      </details>
      <constraints>
        <constraint>バーサーカー因狂化無法使用文字溝通，マスター只能透過令呪精確指揮 [ref:spec-02, section:command-spells]</constraint>
        <constraint>バーサーカー不保有職階技能「対魔力」——即使原始英靈的其他職階有此技能，以バーサーカー召喚時不適用</constraint>
      </constraints>
    </section>

    <section id="stat-generation">
      <title>五圍骰定</title>
      <description>
        サーヴァント的五項能力值（筋力、敏捷、幸運、魔力、耐久）使用千夜月姬 TRPP 聖杯戰爭規則的骰定方式生成。
        骰定在系統分配英靈時自動完成，結果只有從者玩家自己看到。
      </description>
      <details>
        **步驟一——基礎等級**：擲 2D6
        | 骰值 | 等級 |
        |------|------|
        | 3 以下 | E |
        | 4–6 | D |
        | 7–9 | C |
        | 10 | B |
        | 11 | A |
        | 12 | EX |

        **步驟二——正負修正**：若非 EX，再擲 1D6
        | 骰值 | 修正 |
        |------|------|
        | 1 | 獲得「-」|
        | 2–4 | 無修正 |
        | 5 以上 | 獲得「+」|

        **步驟三——修正幅度**：獲得 + 或 - 時，擲 1D6
        | 骰值 | 幅度 |
        |------|------|
        | 1–3 | 一個（+ 或 -）|
        | 4–5 | 兩個（++ 或 --）|
        | 6 | 三個（+++ 或 ---）|

        **情報揭露：**
        - 從者玩家看到自己的完整五圍數值
        - マスター透過契約自動感知五圍等級（A~E，不含 +/- 修正）[ref:spec-04, section:intra-pair]
        - 從者可以謊報 +/- 修正幅度，但基礎等級由契約揭露
      </details>
      <constraints>
        <constraint>EX 等級不進行步驟二、三（無正負修正）</constraint>
        <constraint>所有骰定在伺服器端執行，防止作弊 [ref:spec-08, section:dice-engine]</constraint>
      </constraints>
    </section>

    <section id="stat-effects">
      <title>五圍對戰鬥的影響</title>
      <description>各能力等級對戰鬥數值的具體加成。</description>
      <details>
        | 能力 | 效果 | E | D | C | B | A | EX |
        |------|------|---|---|---|---|---|-----|
        | 筋力 | 傷害加成 | 0 | +1 | +2 | +3 | +4 | 特殊 |
        | 敏捷 | 命中加成 | 0 | +1 | +2 | +3 | +4 | 特殊 |
        | 幸運 | 迴避加成 | 0 | +1 | +2 | +3 | +4 | 特殊 |
        | 魔力 | TP 加成 | 0 | +10 | +20 | +30 | +40 | 特殊 |
        | 耐久 | HP 加成 | 0 | +10 | +20 | +30 | +40 | 特殊 |

        修正規則：
        - 帶「+」時數值翻倍（例如 D+ 的傷害加成為 +2）
        - 帶「-」時數值減半
        - E+ 以最低有效值計算
      </details>
    </section>

    <section id="personal-skills">
      <title>保有スキル</title>
      <description>每騎サーヴァント的個體固有技能。</description>
      <details>
        **技能數量**（擲 1D6）：
        | 骰值 | 技能數 |
        |------|--------|
        | 1–2 | 1 個 |
        | 3–4 | 2 個 |
        | 5 | 3 個 |
        | 6 | 4 個 |

        **技能 Rank**：使用與五圍相同的骰定方式決定（步驟一～三）。

        **技能分配**：由系統從預設技能池中隨機分配，並根據職階進行加權。

        **情報揭露**：保有スキル名稱與效果只有從者玩家自己知道。從者可選擇向マスター揭露或隱瞞。
      </details>
      <examples>
        <example>セイバー較容易獲得劍術相關的保有スキル</example>
        <example>アサシン較容易獲得隱匿、暗殺相關的保有スキル</example>
      </examples>
    </section>

    <section id="noble-phantasm">
      <title>宝具</title>
      <description>
        每騎サーヴァント擁有一個宝具。宝具的真名在遊戲開始時對所有其他玩家隱藏（包括自己的マスター）。
      </description>
      <details>
        **宝具等級與威力上限**：
        | 等級 | 威力上限 |
        |------|---------|
        | E | 10 以下 |
        | D | 20 以下 |
        | C | 30 以下 |
        | B | 40 以下 |
        | A | 50 以下 |
        | 帶 + | 威力翻倍 |

        **宝具類型**：不一定是戰鬥用途，也可能是情報型、結界型、或特殊效果型。

        **真名解放機制**：
        - **決定權在從者玩家**——從者玩家決定是否在戰鬥中宣告宝具真名
        - **宣告真名**（真名解放）→ 發揮完整威力，但真名被在場所有人得知 [ref:spec-04, section:visibility-levels]
        - **不宣告真名** → 只能發揮部分效果，但保密
        - マスター可以「建議」從者解放或不解放，但從者可以無視
        - マスター可用令呪**強制**從者解放真名（消耗 1 畫）或**禁止**解放（消耗 1 畫）[ref:spec-02, section:command-spells]
      </details>
      <constraints>
        <constraint>宝具真名一旦被在場者得知，該情報永久記錄在對應玩家的情報層級中</constraint>
        <constraint>令呪極限突破（三枚）可強制發動宝具隱藏效果 [ref:spec-02, section:command-spells]</constraint>
        <constraint>從者可以選擇何時向マスター揭露宝具資訊——這是組內信任的重要指標</constraint>
      </constraints>
    </section>

    <section id="servant-elimination">
      <title>從者被消滅後</title>
      <description>
        當サーヴァント的 HP 歸零且無法續戰時，從者角色被消滅。
        從者玩家不會直接離開遊戲，而是轉為「英靈殘響」模式。
      </description>
      <details>
        **英靈殘響模式：**
        - 從者玩家可以觀看全場戰況（觀戰視角）
        - 每夜可以向**一位**存活玩家發送一條簡短訊息（上限 50 字）
          - 訊息來源顯示為「英靈殘響」，不顯示具體英靈身份
          - 這代表英靈座中殘留的意念回響
        - 不能使用任何其他通訊頻道
        - 不能移動、不能戰鬥、不能交易

        **特殊情況：**
        - 若場上有失去從者的マスター，英靈殘響模式的從者玩家可以嘗試**再締約**——重新進入遊戲 [ref:spec-07, section:re-contract]
      </details>
    </section>
  </rules>

  <edge_cases>
    <case>若五圍骰定結果全部為 E 或 E-，啟用弱サーヴァント補償機制 [ref:spec-09, section:random-balance]</case>

    <case>宝具為情報型或結界型時，「威力上限」改為「效果強度上限」</case>
    <case>從者對マスター謊報數值時，戰鬥結算仍以實際數值計算——マスター會在戰後從結果推測從者是否說謊</case>
    <case>狂職玩家的「暴走」選項若觸發，可能攻擊自己的マスター——這是狂化的風險</case>
  </edge_cases>
